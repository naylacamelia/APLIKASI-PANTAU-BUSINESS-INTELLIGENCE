<p align="justify">
Pada aplikasi PANTAU (Platform Analitik & Tata Kelola UMKM), proses ETL (Extract, Transform, Load) merupakan tahapan utama dalam membangun sistem Business Intelligence dan data warehouse. Proses ini digunakan untuk mengintegrasikan berbagai data operasional UMKM agar menjadi lebih terstruktur, konsisten, dan siap digunakan dalam proses analisis bisnis serta visualisasi dashboard analytics.Data yang digunakan berasal dari beberapa dataset retail yang mencakup transaksi penjualan, data produk, pelanggan, inventory, logistik, promosi, dan seasonal planning. Seluruh data tersebut diproses menggunakan Python dan library Pandas melalui tiga tahapan utama, yaitu extract, transform, dan load.
</p>

## 📈 𝟐.𝗧𝗿𝗮𝗻𝘀𝗳𝗼𝗿𝗺 (𝗧𝗿𝗮𝗻𝘀𝗳𝗼𝗿𝗺𝗮𝘀𝗶 𝗗𝗮𝘁𝗮)

<p align="justify">
Tahap <b>Transform</b> dilakukan untuk membersihkan dan mengolah data mentah menjadi data yang lebih konsisten, terstruktur, dan siap digunakan dalam proses <i>Business Intelligence</i> serta pembentukan data warehouse pada aplikasi <b>PANTAU</b>. Pada tahap ini dilakukan beberapa proses penting seperti data cleaning, parsing tanggal, konversi tipe data, dan validasi data agar hasil analisis menjadi lebih akurat dan mudah digunakan dalam dashboard analytics.
</p>

---


### 🗃️ 1. Data Understanding dan Data Profiling 

Tahap ini dilakukan untuk memahami kondisi awal dataset sebelum transformasi. Pada tahap ini dilakukan pengecekan terhadap 
dataset kotor untuk menghitung jumlah baris, jumlah kolom, jumlah baris duplikat, dan total nilai kosong. 


### 🧹 2. Penanganan Nilai Duplikat

Tahap ini dilakukan untuk menghapus data duplikat agar tidak terjadi perhitungan ganda pada proses analisis penjualan, inventory, customer, maupun logistik.

✨ Proses ini membantu:
- Menghapus duplicate data
- Mengurangi inkonsistensi data
- Meningkatkan akurasi analisis

---

### 📅 3. Parsing Kolom Tanggal

Kolom tanggal dikonversi ke format datetime agar dapat digunakan untuk analisis berbasis waktu seperti tren penjualan dan forecasting.

📊 Digunakan untuk:
- Tren harian
- Tren bulanan
- Analisis tahunan
- Forecasting penjualan

---

### 🔢 4. Konversi Tipe Data Numerik

Tahap ini dilakukan untuk memastikan kolom numerik dapat digunakan dalam proses perhitungan dan analisis data.

📈 Kolom numerik digunakan untuk:
- Perhitungan revenue
- Analisis inventory
- Forecasting penjualan
- Monitoring performa bisnis

---

### 📝 5. Pembersihan Kolom Teks

Data teks dibersihkan dari nilai kosong dan spasi berlebih agar lebih konsisten saat digunakan dalam proses analisis.


✨ Tujuan proses ini:
- Menghindari data kosong
- Membersihkan format teks
- Menjaga konsistensi data
- Mempermudah proses analisis

----



### 🗂️ 6. Pembentukan Tabel Dimensi

<p align="justify">
Tabel dimensi digunakan untuk menyimpan informasi deskriptif yang menjadi sudut pandang analisis dalam data warehouse. Pada proyek ini dibentuk beberapa tabel dimensi seperti dimensi produk, customer, lokasi, promosi, dan waktu agar data dapat dianalisis berdasarkan produk, pelanggan, cabang, promosi, maupun periode tertentu.
</p>

📦 Dimension Table:
- `dim_product`
- `dim_customer`
- `dim_site`
- `dim_promotion`
- `dim_date`

✨ Tabel dimensi digunakan untuk:
- Analisis produk
- Analisis customer
- Monitoring cabang
- Analisis promosi
- Analisis berbasis waktu


----

### 📊 7. Pembentukan Tabel Fakta

<p align="justify">
Tabel fakta merupakan tabel utama dalam data warehouse yang digunakan untuk menyimpan data aktivitas bisnis dan metrik numerik yang dapat dihitung. Pada proyek PANTAU, tabel fakta digunakan untuk menyimpan data penjualan, inventory, logistik, promosi, dan seasonal planning.
</p>

📦 Fact Table:
- `fact_sales`
- `fact_inventory`
- `fact_logistics`
- `fact_seasonal_planning`

----

### 📈 8. Pembentukan Data Mart

<p align="justify">
Data mart merupakan tabel ringkasan hasil olahan dari data warehouse yang digunakan sebagai sumber utama dashboard analytics pada aplikasi <b>PANTAU</b>. Data mart dibentuk agar proses visualisasi data dan analisis bisnis dapat dilakukan dengan lebih cepat, terstruktur, dan sesuai kebutuhan setiap dashboard.
</p>

📊 Data Mart:
- `owner_kpi_summary`
- `sales_monthly_trend`
- `product_performance`
- `inventory_monitoring_summary`
- `customer_analytics_summary`
- `promotion_performance`
- `logistics_summary`
- `seasonal_planning_summary`

## 📤 𝟯. 𝗟𝗼𝗮𝗱 (𝗠𝗲𝗺𝘂𝗮𝘁 𝗗𝗮𝘁𝗮 𝗸𝗲 𝗗𝗮𝘁𝗮 𝗪𝗮𝗿𝗲𝗵𝗼𝘂𝘀𝗲)


<p align="justify">
Load merupakan tahap akhir dalam proses ETL yang bertujuan untuk menyimpan data hasil transformasi ke dalam bentuk output yang siap digunakan pada dashboard Business Intelligence aplikasi <b>PANTAU</b>.
</p>

<p align="justify">
Pada tahap ini, data disimpan ke dalam dua kelompok utama, yaitu <b>data warehouse</b> dan <b>data mart</b>. Tabel data warehouse terdiri dari tabel dimensi dan tabel fakta yang digunakan sebagai struktur utama penyimpanan data, sedangkan data mart digunakan sebagai sumber visualisasi dashboard analytics.
</p>

📦 Data Warehouse:
- Tabel dimensi (`dim_date`, `dim_product`, `dim_customer`, `dim_site`, `dim_promotion`)
- Tabel fakta (`fact_sales`, `fact_inventory`, `fact_logistics`, `fact_seasonal_planning`)



<p align="justify">
  
Seluruh hasil ETL kemudian diekspor ke dalam file CSV agar dapat digunakan kembali sebagai sumber data dashboard analytics pada aplikasi <b>PANTAU</b>. Dengan tahap load ini, data yang sebelumnya masih berupa data mentah telah berubah menjadi data yang lebih terstruktur, terintegrasi, dan siap digunakan dalam proses analisis serta pengambilan keputusan bisnis.
</p>

## 🛢️ 𝐃𝐚𝐭𝐚 𝐖𝐚𝐫𝐞𝐡𝐨𝐬𝐮𝐞


⭐ Star Schema

<p align="justify">
Data warehouse pada aplikasi PANTAU menggunakan pendekatan Star Schema untuk mendukung proses analisis data dan visualisasi dashboard analytics. Struktur ini terdiri dari tabel fakta sebagai pusat data transaksi yang terhubung dengan tabel dimensi seperti produk, customer, lokasi, promosi, dan waktu.
</p>

<p align="justify">
Pendekatan Star Schema dipilih karena lebih sederhana, mudah dipahami, dan mempercepat proses query analytics pada dashboard Business Intelligence. Selain <b>fact_sales</b>, sistem juga memiliki tabel fakta tambahan seperti inventory, logistic, dan seasonal planning untuk mendukung monitoring operasional dan forecasting bisnis.
</p>

## 💡 𝗜𝗻𝘀𝗶𝗴𝗵𝘁 𝗕𝗶𝘀𝗻𝗶𝘀

<p align="justify">
Aplikasi <b>PANTAU</b> tidak hanya berfungsi sebagai media monitoring data, tetapi juga membantu menghasilkan insight bisnis yang dapat digunakan sebagai pendukung pengambilan keputusan. Melalui proses Business Intelligence, data operasional yang sebelumnya tersebar di berbagai bagian bisnis diolah menjadi informasi yang lebih terstruktur, informatif, dan mudah dipahami melalui dashboard analytics.
</p>

<p align="justify">
Sistem mampu menampilkan berbagai insight penting seperti perkembangan revenue bisnis, performa produk, kondisi inventory, aktivitas logistik, efektivitas promosi, hingga analisis perilaku customer. Seluruh informasi tersebut divisualisasikan dalam bentuk grafik, KPI, dan ringkasan analytics sehingga owner dapat memahami kondisi bisnis secara lebih cepat dan real-time.
</p>

<p align="justify">
Selain membantu proses monitoring, insight bisnis pada aplikasi PANTAU juga dapat digunakan untuk mengetahui tren penjualan, mengevaluasi performa cabang, menentukan produk dengan penjualan terbaik, mengidentifikasi stok yang perlu direstock, hingga membantu proses forecasting dan perencanaan bisnis berdasarkan data historis yang tersedia.
</p>



✨ Insight yang dihasilkan meliputi:
- Monitoring total revenue dan net revenue
- Mengetahui produk terlaris dan performa produk
- Analisis tren penjualan berdasarkan periode tertentu
- Monitoring stok dan inventory cabang
- Analisis perilaku customer dan customer terbaik
- Monitoring efektivitas promosi dan diskon
- Monitoring aktivitas logistik dan pengiriman
- Forecasting penjualan dan seasonal planning
